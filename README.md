# BYD Yard Control — ระบบบริหารลานจอดรถ

ระบบบริหารลานจอดรถยนต์ที่ผลิตออกจากโรงงาน BYD ตั้งแต่ขึ้นหางเทรลเลอร์ → ตรวจรับเข้าลาน (Gate-In) → จัดจอดอัตโนมัติ/กึ่งอัตโนมัติ → ยืนยันจอดสำเร็จ พร้อมแผนผังลาน (Yard Plan), รายการรถ (Unit List) และการสแกน VIN — แนวคิดปรับมาจาก **RoRo TOS**

> A control-room style yard-management system for factory-outbound BYD vehicles. Single-page app, runs locally, data persists in the browser (localStorage). UI เป็นสองภาษา ไทย/อังกฤษ (สลับได้มุมขวาบน).

---

## ▶️ การติดตั้งและรัน / Run

```bash
cd byd-yard
npm install
npm run dev        # เปิด http://localhost:5173
```

ครั้งแรกที่เปิด ระบบจะ **โหลดข้อมูลตัวอย่าง** (20 หาง / ~180 คัน) ให้อัตโนมัติ เพื่อให้เห็นทั้งระบบทำงานทันที — ล้างได้ที่หน้า *นำเข้าข้อมูล*

```bash
npm run build      # build สำหรับ production → ./dist
npm run preview    # ลองรัน build
```

## 🤖 หัวใจของระบบ: Parking Rules Engine

ไฟล์ [`src/lib/parkingEngine.ts`](src/lib/parkingEngine.ts) — เมื่อพนักงานขับสแกน VIN ระบบจะหาช่องจอดที่ดีที่สุดให้ตามกฎ

**กฎต่อรุ่น (ParkingPolicy):**
- `allowedBlocks` — บล็อกที่จอดได้ (เจาะจง เช่น `['A']` หรือ `'ALL'` ทุกบล็อก)
- `rowFrom` / `rowTo` — จำกัดช่วงแถวภายในบล็อก
- `exclusiveRow` — ห้ามปนรุ่นอื่นในแถวเดียวกัน (hard constraint)

**ตัวอย่างที่ตั้งไว้ (แก้ได้ที่หน้า "ตั้งค่ากฎจอด"):**
- **ATTO 3** → บล็อก **A** เท่านั้น แถว **1–10**
- **DOLPHIN** → ทุกบล็อก แต่ **ห้ามปนรุ่นในแถว**
- (รุ่นอื่น ๆ ตั้งตัวอย่างไว้ครบ — SEAL→B,C / SEALION 7→E,F ห้ามปน / HAN→D ฯลฯ)

**ลำดับการเลือกช่อง (scoring):** ต่อแถวรุ่นเดียวกันที่มีที่ว่าง → เปิดแถวว่างใหม่ → (ทางเลือกสุดท้าย) แทรกแถวที่มีรุ่นอื่น — ทำให้รถรุ่นเดียวกันถูกจัดอยู่แถวเดียวกันเป็นระเบียบ

- **Auto plan** — ระบบเลือกให้อัตโนมัติเมื่อสแกน
- **Semi plan** — ระบบเสนอตำแหน่ง แต่กด"เลือกตำแหน่งเอง" เพื่อคลิกช่องว่างบนผังบล็อกได้

> ✔️ ตรวจสอบแล้วกับข้อมูลตัวอย่าง: ATTO 3 ทุกคันอยู่บล็อก A แถว 1–10 (0 ผิดกฎ), ไม่มีแถวไหนที่ DOLPHIN ปนรุ่นอื่น

## ✨ ลูกเล่นเพิ่มเติม

- **⌘K Command Palette** — ค้นหา VIN/รุ่น หรือกระโดดไปเมนูได้ทันที
- **จัดจอดอัตโนมัติทั้งหมด** — ปุ่มเดียววางแผนจอดรถที่รออยู่ทั้งหมดตามกฎ (หน้า Yard Plan)
- **แผนผังบล็อกแบบ heatmap** — สีตามรุ่น เหมือน RoRo TOS, คลิกดูรายช่อง
- **การ์ดรถ** สไตล์ใบตรวจ BYD + ภาพรถมุมบน, **ไทม์ไลน์** การเคลื่อนไหวต่อคัน
- **บันทึกอัตโนมัติ** ทุกอย่างลง localStorage — ปิดเปิดใหม่ข้อมูลอยู่ครบ
- สลับ **ไทย/อังกฤษ** และ **Auto/Semi** ได้จาก topbar

## 📸 เก็บรูปตำหนิบน Cloudflare R2

รูปถ่ายตำหนิจะถูกอัปโหลดขึ้น **R2** (ผ่าน Pages Function `functions/api/photos/`)
แล้วเก็บแค่ URL สั้น ๆ ในฐานข้อมูล — ฐานข้อมูล Supabase ไม่บวมด้วย base64 อีกต่อไป
ถ้ายังไม่ได้ตั้งค่า R2 ระบบจะ fallback เก็บรูปแบบ data-URL เหมือนเดิมอัตโนมัติ (ไม่มีอะไรพัง)

**ตั้งค่าครั้งเดียวใน Cloudflare dashboard:**
1. **R2 → Create bucket** ชื่อ `sjwd-photos` (ต้องเปิดใช้ R2 ก่อน — มี free tier 10 GB)
2. **Workers & Pages → byd-yard-sjwd → Settings → Bindings → Add → R2 bucket**
   - Variable name: `PHOTOS` · Bucket: `sjwd-photos`
3. Deploy รอบถัดไป (push ขึ้น main) — รูปใหม่จะขึ้น R2 ทันที

รูปเก่าที่เป็น data-URL ยังแสดงผลได้ปกติ ไม่ต้อง migrate

## ⚙️ ค่าเริ่มต้นกลางของ Unit List (คอลัมน์ + ช่องกรอง)

แอดมินตั้ง "คอลัมน์ + ช่องกรอง" ปัจจุบันเป็น **ค่าเริ่มต้นของทุกคน** ได้ (ปุ่มในแผงจัดการคอลัมน์ / ปรับแต่งช่องกรอง → "ตั้งเป็นค่าเริ่มต้นของทุกคน") ค่าถูกเก็บบน Supabase ตาราง `app_config`
- เครื่องใหม่ / ผู้ใช้ใหม่จะเริ่มด้วยค่านี้อัตโนมัติ (seed ครั้งเดียว) แล้วปรับแต่งของตัวเองได้ภายหลัง
- ใครก็กด "โหลดค่าเริ่มต้น" เพื่อดึงค่าล่าสุดที่แอดมินตั้งไว้มาแทนได้

**ตั้งค่าครั้งเดียว:** รัน `supabase-app-config.sql` ใน Supabase Dashboard → SQL Editor
(ถ้ายังไม่รัน ปุ่ม "ตั้งเป็นค่าเริ่มต้น" จะแจ้ง error — ส่วนอื่นของแอปทำงานปกติ)
